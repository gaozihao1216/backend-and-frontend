# 类型、系统架构与功能简介

本项目是一个以 Angry Birds 类玩法为灵感的全栈 UGC 关卡平台。系统围绕“设计师创作关卡、管理员审核发布、玩家游玩评价”这一主流程展开，并扩展了总监管理、UI 定制、鸟类设计、玩家商店与社交等能力。

## 类型与系统架构

- **前端类型体系**：前端采用 React + TypeScript + Vite，开启严格类型检查，并使用 Zod 对 API 请求与响应数据做运行时校验。
- **后端类型体系**：后端采用 Scala 3 + http4s + cats-effect + Circe，通过强类型领域对象、`Either[HttpError, A]` 和统一 APIMessage 模式组织业务逻辑。
- **前后端契约**：前端 `frontend/src/api/` 与后端 `backend/microservice/src/*/api/` 采用同名 API 文件和对齐的数据对象，降低接口漂移风险。
- **系统分层**：前端按页面、组件、Hook、API、对象模型拆分；后端按 user、level、admin、ui、bird、player 等业务域拆分，路由只负责解析请求，业务逻辑下沉到 APIMessage 与表/存储层。
- **存储模式**：默认使用内存存储便于演示与开发，也支持 PostgreSQL/JDBC 持久化。

## 主要功能

- **设计师端**：关卡创建、可视化编辑、地形与实体配置、草稿保存、提交审核、自定义鸟类设计。
- **管理员端**：关卡审核、评论管理、鸟类设计审核。
- **总监端**：管理员权限控制、页面与按钮 UI 定制、地图与关卡槽位配置、鸟类技能实验功能。
- **玩家端**：地图选关、物理引擎游玩、评分、收藏、评论、商店、社交与备战页面。
- **基础系统**：模拟登录、后端用户绑定、角色权限控制、统一 API 响应格式、前端代理与本地开发脚本。

## 特点与亮点

- **类型安全的全栈契约**：TypeScript/Zod 与 Scala/Circe 双侧约束接口数据，减少前后端联调错误。
- **清晰的领域模块化**：关卡、玩家、管理员、鸟类、UI 定制等能力按业务域隔离，便于扩展和维护。
- **统一事务与错误处理**：后端通过 APIMessage 统一进入事务，成功提交、失败回滚，并统一返回错误结构。
- **可演示的完整闭环**：从设计师创建关卡，到管理员审核，再到玩家游玩评分，形成可运行的 UGC 内容流转链路。
- **可扩展的管理能力**：总监角色可以调整页面、按钮、地图槽位等配置，为后续低代码运营配置提供基础。
- **游戏化前端体验**：结合 Matter.js 物理引擎与可视化编辑器，让平台不仅是 CRUD 系统，也具备实际玩法展示能力。
